    One-Page Quick Reference Cards

    CARD 1: Starting a New Project

    ☑ NEW PROJECT CHECKLIST (15 minutes)

    1. Create folder: Project_YYYYMM_ShortName
    2. Copy template README.md from /99_DOCUMENTATION/Templates/
    3. Fill in README.md (project title, PI, description, data types)
    4. Copy template metadata.yaml
    5. Fill in metadata.yaml (at minimum: project_id, pi, start_date)
    6. Create subdirectories:
    7. 01_raw_data/ (with data type folders)
    8. 02_processed/
    9. 03_analysis/
    10. 04_figures/
    11. Add project to lab inventory: /99_DOCUMENTATION/project_inventory.xlsx
    12. Notify Data Steward via email

    Time investment: 15 minutes
    Prevents: Hours of future confusion and data loss

    CARD 2: Saving New Data

    ☑ NEW DATA CHECKLIST (5 minutes per dataset)

    1. Save to correct location:
    2. Imaging → 01_raw_data/imaging/
    3. IMC → 01_raw_data/imc/
    4. scRNA-seq → 01_raw_data/scrnaseq/

    5. Spatial → 01_raw_data/spatial/

    6. Use naming convention: YYYYMMDD_DataType_Sample_Condition.ext

    7. Generate checksum: 

      sha256sum myfile.tif > myfile.tif.sha256

    8. Update metadata.yaml:

    9. Add new dataset entry

    10. Record date, sample info, experimental conditions

    11. Backup will run automatically (weekly)

    Time investment: 5 minutes
    Prevents: Data corruption, lost samples, cannot reproduce results

    CARD 3: Monthly Data Hygiene

    ☑ MONTHLY CHECKLIST (1 hour per month)

    Week 1: - [ ] Run storage monitor: python storage_monitor.py /RDS - [ ] Review storage report - [ ] If >80% full, proceed to emergency cleanup

    Week 2: - [ ] Compress imaging data >3 months old - [ ] Expected recovery: 100-200GB - [ ] Run: python compress_old_imaging.py --days 90

    Week 3: - [ ] Move projects >6 months inactive to 01_ARCHIVED_PROJECTS/ - [ ] Delete temporary analysis files (cache/, *.tmp) - [ ] Find: find /RDS -name "cache" -type d -exec rm -rf {} \;

    Week 4: - [ ] Verify backup integrity (spot check 5 random files) - [ ] Rotate external HDD (swap onsite/offsite) - [ ] Update lab data inventory

    Time investment: 1 hour per month
    Prevents: Storage crises, data loss, backup failures

    metadata.yaml Template (Required for all projects)

    # Project Metadata
project_id: "Project_YYYYMM_Name"
pi_name: "Dr. Name"
pi_email: "pi@university.edu"
lead_researcher: "Researcher Name"
start_date: "YYYY-MM-DD"
status: "active"  # active, archived, published
funding_source: "Grant XYZ"

# Data Summary

  - imaging
  - imc
  - scrnaseq
  - spatial

total_size_gb: 0  # Update periodically

# Raw Data Inventory

    date_acquired: "2024-01-15"

    modality: "confocal"
    sample_type: "mouse_brain"
    size_gb: 50

    notes: "Initial pilot experiment"

  - date: "2024-01-20"

    software: "CellProfiler 4.2"
    output: "02_processed/imaging/20240120_segmented/"

# Publications
publications: []

# Last Updated
last_updated: "YYYY-MM-DD"
